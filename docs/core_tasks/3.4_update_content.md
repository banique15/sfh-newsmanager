# Update Existing Content

## Overview

User asks to change an existing article (e.g., "Update the summer event article with the new date", "Change the title to…", "Set the content to…", "Update it with…").

Updates can overwrite live content, so we must identify the right article and get explicit approval before making changes.

## Why This Approach

Updates can overwrite live content, so we must change only the right article and only after explicit approval to prevent accidental data loss.

## Prerequisites

- **Required tools:**
  - `intent_detection` - Parse what to update and how
  - `context_resolution` - Resolve "it", "that one" to article ID
  - `article_search` - Find article if specified by title/topic
  - `article_crud` - Perform update operation
  - `yes_no_prompt` - Get approval
  - `conversation_memory` - Track updated article
  
- **Required context:**
  - Which article (by title, ID, or reference)
  - What to change (title, content, excerpt, image, etc.)

## Step-by-Step Procedure

### 1. **Understand What to Update and How**

**Action:** Use user's words and conversation context to identify which article and what should change.

**Tool(s) used:** `intent_detection`, `context_resolution`

**Identify:**
- **Target article:** "summer event article", "it", "that one", "the gala recap"
- **What to change:** body, title, excerpt, hero_image_url, etc.
- **New value:** the replacement content

**Resolution strategies:**
1. **Explicit title** - "Update 'Summer Concert 2026'..."
2. **Topic reference** - "Update the gala article..." → search for gala
3. **Context reference** - "Update it" → check conversation memory
4. **Link provided** - See [4.4_user_provides_link.md](../edge_cases/4.4_user_provides_link.md)

**If ambiguous:** Follow [4.2_multiple_matches.md](../edge_cases/4.2_multiple_matches.md)

**If unclear:** Follow [3.10_clarification.md](3.10_clarification.md)

---

### 2. **Propose the Change in Plain Language**

**Action:** State exactly what will change and offer Yes/Cancel.

**Tool(s) used:** `yes_no_prompt`

**Message format:**
```
I'll update '[ARTICLE_TITLE]' so the [field] is '[new value]'.

[Show before/after if helpful]

Is this correct?
```

**Examples:**
> "I'll update 'Summer Concert Series 2026' so the content includes the new date (June 20 instead of June 15). Is this correct?"

> "I'll change the title from 'Gala Event' to 'Annual Fundraising Gala 2026'. Is this correct?"

**Options:** [Yes] [Cancel]

---

### 3. **Apply Only After Confirmation**

**Action:** If they confirm, update only that article. If they cancel, do nothing.

**Tool(s) used:** `article_crud`

**On Yes:**
```json
{
  "operation": "update",
  "article_id": "[resolved ID]",
  "title": "[new value if changing title]",
  "content": "[new value if changing content]",
  "excerpt": "[new value if changing excerpt]",
  "hero_image_url": "[new value if changing image]"
}
```

**Only update specified fields** (partial update, don't overwrite others)

**On Cancel:** Follow [3.12_user_cancels.md](3.12_user_cancels.md)

---

### 4. **Confirm the Result**

**Action:** Tell them the content was updated or explain failure.

**Success message:**
```
✅ Updated '[ARTICLE_TITLE]'.

[Link to article if available]
[Next steps if applicable]
```

**Example:**
> "✅ Updated 'Summer Concert Series 2026' with the new date. The article is still a draft. Let me know if you'd like to publish it!"

**Store in memory:** Updated article for potential follow-ups

---

## Expected Outcome

User approves exactly what changes; only then does the article update. They receive clear confirmation of what was changed.

## Error Handling

| Error Scenario | Response |
|---------------|----------|
| **Can't identify article** | "Which article should I update?" List options if multiple match. (see 4.2) |
| **Multiple matches** | "Which one? 1. [title] 2. [title]..." (see 4.2) |
| **Article not found** | "I couldn't find that article. Could you specify the title?" |
| **Update failed** | "Something went wrong updating the article. Please try again." |
| **User cancels** | "No problem. The article wasn't changed." (see 3.12) |
