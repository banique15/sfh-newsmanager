# User Sends Content in the Message (Paste)

## Overview

User pastes the full body of an article or a long paragraph and says "use this for the article" or "create an article with this".

We treat the pasted text as the primary content, optionally generate missing parts (headline, excerpt), and confirm before saving.

## Why This Approach

User-provided content should be preserved as-is. We fill in missing structure (title, excerpt) but don't overwrite their text.

## Prerequisites

- **Required tools:**
  - `intent_detection` - Detect pasted content
  - `content_generation` - Generate title/excerpt if needed
  - `article_crud` - Save article
  - `yes_no_prompt` - Confirm before saving

## Step-by-Step Procedure

### 1. **Detect Pasted Content**

**Action:** Recognize when user provides full content in message.

**Tool(s) used:** `intent_detection`

**Indicators:**
- Long text block in message
- Phrases like "use this", "paste this", "create with this content"
- Formatted text (paragraphs, line breaks)

---

### 2. **Use Pasted Text as Content**

**Action:** Treat pasted text as the article body (content field).

**What to preserve:**
- User's exact wording
- Formatting (paragraphs, line breaks)
- Any markdown they included

**What to generate (if missing):**
- Title/headline - extract from first line or generate
- Excerpt - use first 1-2 sentences

**Tool(s) used:** `content_generation` (only for missing parts)

---

### 3. **Confirm What We'll Create**

**Action:** Show what will be saved and ask for approval.

**Tool(s) used:** `yes_no_prompt`

**Message format:**
```
I'll create an article titled '[GENERATED_TITLE]' with the content you pasted.

[Show excerpt or first few lines]

Is this correct?
```

**Example:**
> "I'll create an article titled 'Summer Fundraiser Success' with the content you pasted. Is this correct?"

**Options:** [Yes] [Cancel]

---

### 4. **Save After Confirmation**

**Tool(s) used:** `article_crud`

**Parameters:**
```json
{
  "operation": "create",
  "title": "[generated or extracted]",
  "content": "[user's pasted text]",
  "excerpt": "[generated from pasted text]",
  "status": "draft"
}
```

**Follow:** [3.2_create_article_simple.md](../core_tasks/3.2_create_article_simple.md) confirmation flow

---

## Expected Outcome

Article created with user's exact content, generated title/excerpt, saved as draft after confirmation.
